# 预设系统设计讨论

> 状态：讨论中 | 2026-05-26

---

## 1. 当前问题

`dialogue_system_prompt` 把**世界观/人物**和**写作规则**混在一起：

```
你是一位小说代笔助手，正在协助创作故事《机械神座》。

【世界观】           ← 故事专属，换故事就变
近未来都市...

【主角】             ← 故事专属
陈龙，一名程序员...

【写作规则】         ← 通用，跨故事可复用
1. 第三人称有限视角
2. 用户给情节，AI 扩写
3. 300-600字/次
...
```

问题：换一个故事，写作规则想复用却得重新生成一遍。

---

## 2. 核心设计

### 2.1 预设（Preset）是什么

**预设 = 告诉大模型它应该扮演什么角色。** 本质上是一段角色定义 + 行为规则。

当前「机械神座」这个故事的 `dialogue_system_prompt` 中，预设信息藏在"你是一位小说代笔助手"这句话里——它实际上定义了一个"小说代笔"角色。

| 属于预设 | 不属于预设（故事专属） |
|----------|----------------------|
| 角色定义（代笔/扮演/编辑...） | 故事背景 |
| 写作规则（人称/节奏/字数...） | 世界观 |
| 模式（小说/角色扮演） | 主角设定 |
| 行为约束（视角/结尾格式...） | NPC 设定 |
| | 时间地点 |
| | 故事名字 |

### 2.2 预设与故事的关系

- **一个预设可被无数故事/对话引用** — 全局复用
- **一个故事不绑定预设** — 每个对话独立选择引用哪个预设
- **新建故事 ≠ 新建预设** — 故事是故事，预设是预设，两个独立概念
- **但创建故事时可能触发创建预设** — 如果用户选择的类型没有现成预设

### 2.3 system prompt 的组装方式

```
完整的 system prompt = 【世界观 + 主角 + NPC】（来自 dialogue config）
                      + 【角色定义 + 写作规则】（来自 preset）
```

- 世界观/主角/NPC → 对话创建时确定，**跟随对话，不可变**
- 角色定义 + 写作规则 → 引用某个 preset，**可热切换**

### 2.4 预设库示例

```
预设库（全局，不绑定任何故事）
├── 📝 网络爽文代笔      "快节奏、爽点密集、杀伐果断..."
├── 📝 轻小说代笔        "日常细腻、对话为主、日系风格..."
├── 🎭 沉浸角色扮演      "以 NPC 身份第一人称回应..."
├── ✍️ 传统文学编辑      "文笔厚重、描写细腻..."
└── ...（用户自由创建/编辑/删除）

故事 A 对话 1 ──引用──→ 网络爽文代笔
故事 B 对话 1 ──引用──→ 轻小说代笔
故事 A 对话 2 ──引用──→ 沉浸角色扮演  （同一故事不同对话可用不同预设）
```

### 2.5 创建预设的两种路径

| 路径 | 场景 | 触发方式 |
|------|------|---------|
| **A. 预设管理界面** | 用户主动管理预设库 | 手动点击「新建预设」，填写名称和规则 |
| **B. 创建故事时隐式创建** | 开场白流程中，用户选择的类型没有对应预设 | 智能体问用户期望的写作风格 → 自动生成并保存为预设 |

路径 B 的关键：开场白完成后，生成的写作规则部分自动保存为一个新预设，用户可以选择命名。

## 3. 功能清单

### 3.1 预设 CRUD
- **创建**：
  - 路径 A：预设管理界面手动创建，填写名称 + 角色定义 + 规则
  - 路径 B：创建故事/对话时，智能体根据用户需求自动生成并保存
- **编辑**：修改预设的名称、角色定义、规则文本
- **删除**：删除前警告（如有对话正在引用）
- **列表**：全局预设管理界面，显示名称、模式、引用数

### 3.2 预设热切换
- 对话中切换 preset → system 消息变化 → KV Cache 失效
- 切换后对话历史保留，新的 system 从下一条消息生效
- 入口：DialogueView 菜单 → 切换预设

### 3.3 开场白流程（最终版）

**第一步就是选择预设**，后续再确认故事细节。

```
第 1 步：选择预设
  ├── 已有预设 → 选一个，跳到第 2 步
  └── 创建新预设 → 智能体对话引导：
        ├── 问：小说还是角色扮演？（mode）
        ├── 问：第一人称还是第三人称？（pov）
        ├── 问：希望 AI 扮演什么角色？（role）
        └── 问：有什么写作规则要求？（rules）
        → 生成预设 → 保存到预设库

第 2 步：时间
第 3 步：地点
第 4 步：主角（用户扮演）
第 5 步：NPC（AI 扮演）
→ 调用 start_dialogue（引用选中的 preset）
```

**关键变化**：人称和模式不再单独询问，它们包含在 preset 中。如果用户选了已有 preset，跳过这些。如果新建 preset，智能体按对话方式引导填写。

### 3.4 预设管理入口
- 顶部工具栏预设图标 → 打开预设管理面板（替换主内容区）
- 系统智能体对话中也可以管理（说「管理预设」）

### 3.5 AI 辅助编辑预设
- 在预设管理面板中编辑/新建预设时，可随时召唤系统智能体
- 智能体拿到当前表单内容，通过对话帮助修改（调整角色定义、优化规则措辞等）
- 智能体调用 `update_preset` 后，面板表单自动刷新

## 4. 数据结构草案

### books.json 不变
不存 preset 引用。preset 由对话级别引用。

### dialogue 记录
```json
{
  "id": "dlg_xxx",
  "bookId": 1,
  "presetId": "preset_xxx",
  "config": {
    "time": "近未来都市",
    "place": "公寓",
    "protagonist": { "name": "陈龙", "description": "..." },
    "npcs": []
    // 注意：mode、pov、dialogue_system_prompt 已移除
  },
  "messages": [...]
}
```

### 预设文件 `.data/presets.json`
```json
[
  {
    "id": "preset_xxx",
    "name": "网络爽文代笔",
    "mode": "novel",
    "pov": "third",
    "role": "你是一位小说代笔助手...",
    "rules": "1. 采用第三人称有限视角，始终以主角视角叙述。\n2. 用户给出情节方向，你负责扩写为细腻的叙事段落...\n3. 每次回复 300-600 字...",
    "createdAt": "2026-05-26T..."
  }
]
```

### system prompt 组装
```
system = f"""
{config 生成的世界观/主角/NPC 描述}

---
{preset.role}

{preset.rules}
"""
```

**热切换时**：下一条 system 消息前插入提示：
```
--- 写作风格已切换为「{preset.name}」---
```

## 5. 已决策

- [x] **热切换 preset 时插入提示消息**（方案 B）→ 对话中插入一条 `"--- 写作风格已切换为「{preset_name}」---"` 的系统提示
- [x] **编辑 preset 后自动更新所有引用**（方案 A，引用模式）→ preset 是源头，对话引用它，一改全改
- [x] **「模式」和「人称」放入 preset** → dialogue config 只保留纯故事信息（时间、地点、主角、NPC）
- [x] **创建故事时隐式创建预设**（方案 A）→ 智能体当场引导用户描述风格，生成新预设，顺便完成开场白
